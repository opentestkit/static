OTK Style Guide

Open Test Kit's design system in one page. Copy the import block, use semantic classes and --otk-* tokens, and your UI will look right.

Foundation

Import

Add these four tags in order. Do not change the order or skip a file.

<html lang="en" data-theme="dark">
<head>
  <link rel="stylesheet" href="https://static.opentestkit.org/style/tokens.css">
  <link rel="stylesheet" href="https://static.opentestkit.org/style/base.css">
  <link rel="stylesheet" href="https://static.opentestkit.org/style/components.css">
  <script src="https://static.opentestkit.org/style/components.js" defer></script>
</head>

data-theme="dark" is required on <html>. For light mode, use data-theme="light". The IBM Plex font family loads automatically from base.css — do not add a separate font import.

Rules

Never hard-code colors, spacing, or fonts. Always use --otk-* tokens. No inline style=. Reach for existing classes before writing new CSS.

Typography

Headings

Six levels. h1–h2 use IBM Plex Serif; h3 and h5 use Sans; h4 and h6 use Mono in uppercase.

h1 — IBM Plex Serif light, 2.25 rem

h2 — IBM Plex Serif regular, 1.5 rem

h3 — IBM Plex Sans medium, 1.125 rem

h4 — IBM Plex Mono uppercase, 0.875 rem

h5 — IBM Plex Sans semibold, 0.875 rem
h6 — IBM Plex Mono uppercase, 0.75 rem

Paragraph styles

Lead — IBM Plex Serif light at 1.25 rem. Use for subtitles and introductory sentences. Add class .lead to a <p>.

Body paragraph — IBM Plex Sans at 1 rem with 1.8 line-height. Max-width is capped at --otk-measure (68 ch). Text color is --otk-color-text-secondary rather than pure white for comfortable reading.

Small / caption — mono, 0.75 rem, wide tracking. Use <small> or class .small.

Primary violet #4816FF. Used for interactive elements, focus rings, and emphasis.

Details / Summary

When should I override a token?

Only when the default value doesn't work for your specific use case. Overrides must still use other --otk-* tokens — never raw hex or px.

Can I use Tailwind alongside OTK?

Yes. All OTK tokens are namespaced --otk-* to avoid conflicts with Tailwind v4's --color-*, --font-*, and --text-* variables.
